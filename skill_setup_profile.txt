## Skill Structure Pattern
Skills are self-contained with:
- SKILL.md (description + usage)
- scripts/ (optional CLI tools)
- node_modules/ if npm dependencies needed
- Often installed via ClawHub (`clawhub install <skill>`)

Common script patterns:
- Python scripts use shebang #!/usr/bin/env python3, run from workspace root
- Node scripts often in skills/*/scripts/
- Environment variables passed explicitly (GOG_KEYRING_PASSWORD, USE_SMTP)
- Logging to files in workspace root

## Notes for New Skill Setup
- If skill needs external APIs, store tokens in .github-token.json pattern or as env vars
- Always handle GOG_KEYRING_PASSWORD for Google access
- Use USE_SMTP=1 for email sending
- Cron jobs should set required env explicitly
- Scripts should be executable and have proper shebangs
- Python dependencies: venv at /root/.openclaw/workspace/venv
- Node dependencies: installed per-skill in node_modules
- Playwright browsers: /root/.cache/ms-playwright/
- For PTY/interactive: use exec with pty=true
- For long-running: consider sessions_spawn with runtime="subagent" or "acp"
