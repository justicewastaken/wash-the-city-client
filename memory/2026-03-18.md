## Wash The City Contact Form Fix

**Status:** SOLVED — Deployed and working

### Problem
Form submissions were not being captured by Netlify Forms and the page was redirecting on submit instead of staying in place with inline feedback.

### Initial Approach (Failed)
- Existing code had `data-netlify="true"` and an AJAX handler that intercepted submit events
- Pushed redeploy to get it live
- Still failed because Netlify's built-in form handling interfered: `method="POST"` triggered native submission, and Netlify injects its own handler that could race with custom JS

### Final Solution (WORKED)
**Commit:** `f0cf8e6` — Decouple Netlify detection from visible form; use XHR button click handler

Key changes:
1. **Visible form** has NO Netlify attributes, NO `method="POST"`, no hidden inputs
   - `<form id="contactForm" name="contact" novalidate>`
   - Button is `type="button"` with `onclick="submitForm()"`
2. **Hidden form** exists solely for Netlify detection:
   ```html
   <form name="contact" data-netlify="true" hidden>
     <input type="hidden" name="form-name" value="contact">
     <input type="hidden" name="to" value="justiceforanything@gmail.com">
     <input type="hidden" name="name">
     <input type="hidden" name="phone">
     <input type="hidden" name="email">
     <input type="hidden" name="service">
     <input type="hidden" name="message">
   </form>
   ```
3. **submitForm()** function manually collects data from visible form fields and sends via `XMLHttpRequest` to `/` with `application/x-www-form-urlencoded`. No form submit event is ever triggered, so Netlify cannot intercept or redirect.

**Result:** Form submissions are captured by Netlify, page stays in place, button shows "Your request has been submitted." on success.

### Important Note for Future
- Do NOT add `method="POST"` or `data-netlify` to the visible form when using this pattern
- Keep a separate hidden form with `data-netlify="true"` for Netlify to detect
- Use button `onclick` or event listener directly on button, not form submit
- This is the recommended approach for JS-rendered forms per Netlify docs
