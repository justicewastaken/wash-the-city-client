## Website Sales Automation — Pipeline Built

### Goal
Daily automated outreach to new local salon leads:
- Research (Google Maps scrape)
- Deduplication
- Website enrichment (find emails, booking platforms via Playwright)
- Generate personalized emails
- Send a daily batch

### Components

**Core scripts:**
- `daily_website_batch.py` — Sends a batch (default 20) from `salon_emails_remaining.json`; updates `leads_master.json` status to 'emailed'
- `pipelines/scrape_new_leads.py` — Scrapes Google Maps for new businesses; appends to `salon_leads.csv` (dedupes, adds city)
- `pipelines/run_enrichment.sh` — Runs `enrich_salons_v2.js` (Node/Playwright) to fill emails and booking platforms
- `pipelines/refresh_master_from_enriched.py` — Merges `salon_leads_enriched.csv` into `leads_master.json`
- `pipelines/generate_missing_emails.py` — Creates initial emails for any new leads in master not yet in `salon_emails_remaining.json`

**Orchestration:**
- `full_daily_pipeline.sh` — Runs all steps in sequence

### Data files
- `salon_leads.csv` — Raw leads (accumulates)
- `salon_leads_enriched.csv` — Enriched with emails/booking platforms
- `leads_master.json` — Master list with status tracking
- `salon_emails_remaining.json` — Pool of unsent emails (drafts)

### Environment requirements
- Node.js + Playwright (for `enrich_salons_v2.js`)
- Python3
- Gmail via `gog` (configured)
- Optional: `PORTFOLIO_URL` and `YOUR_PHONE` env vars (defaults already set in scripts)

### Usage

Test first (dry-run):
Edit `full_daily_pipeline.sh` and change the batch step to:
`python3 daily_website_batch.py --dry-run --batch-size 20`

Then run manually:
`./full_daily_pipeline.sh`

Once happy, schedule via cron daily e.g. 9 AM CT:
```
0 14 * * * /root/.openclaw/workspace/full_daily_pipeline.sh >> /root/.openclaw/workspace/pipeline.log 2>&1
```
(14:00 UTC = 9 AM CT)

### Notes
- Enrichment script skips leads that already have an email, so only new raw leads are processed each day.
- Duplicate detection: by business name (case-insensitive, slugified) across master and CSV.
- City field is required for email template; provide city in `scrape_new_leads.py` call or modify script to parse from query.
- Customize email template in `pipelines/generate_missing_emails.py` if needed.

### Known Issues (March 18+)
- **Deliverability:** Bounces and "limit reached" errors indicate invalid/risky emails in send pool. Need to add email validation during enrichment (before adding to `salon_emails_remaining.json`) or adjust sending parameters. This blocks scaling.
